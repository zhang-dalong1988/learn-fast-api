# 响应模型

## 1. 声明响应模型

你可以使用路径操作装饰器中的 `response_model` 参数来声明用于响应的模型。

```python
# 导入 FastAPI 和 BaseModel
from fastapi import FastAPI
from pydantic import BaseModel

# 创建应用实例
app = FastAPI()


# 定义 Item 数据模型
class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None
    tags: list[str] = []


# 定义用户输入模型, 包含密码字段
class UserIn(BaseModel):
    username: str
    password: str  # 密码字段
    full_name: str | None = None
    email: str | None = None


# 定义用户输出模型, 不包含密码字段
class UserOut(BaseModel):
    username: str
    full_name: str | None = None
    email: str | None = None


# 定义 POST 路径操作, 使用 response_model 声明响应模型
@app.post("/user/", response_model=UserOut)
async def create_user(user: UserIn):
    # 返回 user, 但只包含 UserOut 中定义的字段
    # password 字段会被自动排除
    return user
```

## 2. 返回相同的输入数据

在这个例子中, 模型 `UserIn` 有一个 `password` 字段, 但我们不想在响应中返回它。

通过使用 `response_model=UserOut`, FastAPI 会:
* 将数据转换为 `response_model` 中声明的模型
* 验证数据
* 将它添加到 OpenAPI 模式中
* 在自动文档 UI 中使用它

## 3. 响应模型编码

你的响应模型可以有默认值, 例如:

```python
# 导入 FastAPI 和 BaseModel
from fastapi import FastAPI
from pydantic import BaseModel

# 创建应用实例
app = FastAPI()


# 定义 Item 数据模型, 包含默认值
class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float = 10.5  # 默认税率为 10.5
    tags: list[str] = []  # 默认标签为空列表


# 模拟数据库
items = {
    "foo": {"name": "Foo", "price": 50.2},
    "bar": {"name": "Bar", "description": "The bartenders", "price": 62, "tax": 20.2},
    "baz": {"name": "Baz", "description": None, "price": 50.2, "tax": 10.5, "tags": []},
}


# 定义 GET 路径操作, 使用 response_model_exclude_unset=True
@app.get(
    "/items/{item_id}",
    response_model=Item,
    response_model_exclude_unset=True,  # 排除未设置的字段
)
async def read_item(item_id: str):
    # 返回对应的项目
    return items[item_id]
```

## 4. 总结

使用 `response_model` 你可以:

* 声明响应模型
* 排除未显式设置的字段
* 使用不同的输入和输出模型

以及更多!
