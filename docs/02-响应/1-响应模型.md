# 响应模型

你可以在任何路径操作中声明用于响应的 `response_model`.

## 使用 `response_model` 参数

你可以声明用于响应的模型:

```python
from typing import Any

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float = 10.5
    tags: list[str] = []

items = {
    "foo": {"name": "Foo", "price": 50.2},
    "bar": {"name": "Bar", "description": "The bartenders", "price": 62, "tax": 20.2},
    "baz": {"name": "Baz", "description": None, "price": 50.2, "tax": 10.5, "tags": []},
}

@app.get("/items/{item_id}", response_model=Item)
async def read_item(item_id: str):
    return items[item_id]
```

## `response_model` 参数的好处

FastAPI 将使用此 `response_model` 来:

- 将输出数据转换为其类型声明
- 验证数据
- 在 OpenAPI schema 中添加 JSON Schema 用于响应
- 被自动文档 UI 使用

## 它的主要作用是什么

它的主要作用是:

### 限制输出数据

即使你的函数返回的内容包含更多数据,响应模型也只会包含在响应模型中声明的数据.

```python
from typing import Any

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class UserIn(BaseModel):
    username: str
    password: str
    email: str
    full_name: str | None = None

class UserOut(BaseModel):
    username: str
    email: str
    full_name: str | None = None

# 模拟数据库
fake_users_db = {}

@app.post("/user/", response_model=UserOut)
async def create_user(user: UserIn):
    user_dict = user.model_dump()
    # 密码应该被散列存储
    user_dict["hashed_password"] = "fakehashed" + user_dict.pop("password")
    fake_users_db[user.username] = user_dict
    return user_dict
```

## 将响应模型用作输入模型

在上面所有例子中,我们使用不同的模型来定义输入和输出.

虽然不总是必需的,但这是一个很好的实践,特别是因为安全原因.

在上面的例子中,用户在输入中发送了 `password`,但输出中不会包含它.

另外,我们创建了一个包含散列密码的 `user_dict`,但响应中不会包含它.

## 返回相同的输入数据

有时你需要将输入数据返回,但使用不同的模型.

例如,在更新数据时.

你可以使用 `response_model_exclude_unset=True` 来返回只设置了显式值的数据:

```python
from typing import Any

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float = 10.5
    tags: list[str] = []

items = {
    "foo": {"name": "Foo", "price": 50.2},
    "bar": {"name": "Bar", "description": "The bartenders", "price": 62, "tax": 20.2},
    "baz": {"name": "Baz", "description": None, "price": 50.2, "tax": 10.5, "tags": []},
}

@app.patch("/items/{item_id}", response_model=Item, response_model_exclude_unset=True)
async def update_item(item_id: str, item: Item):
    stored_item_data = items[item_id]
    stored_item_model = Item(**stored_item_data)
    update_data = item.model_dump(exclude_unset=True)
    updated_item = stored_item_model.model_copy(update=update_data)
    items[item_id] = updated_item.model_dump()
    return updated_item
```

## 其他响应模型参数

你还可以使用:

- `response_model_exclude`:从响应中排除某些字段
- `response_model_include`:只包含响应中的某些字段

```python
from typing import Any

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float = 10.5

items = {
    "foo": {"name": "Foo", "price": 50.2},
    "bar": {"name": "Bar", "description": "The Bar fighters", "price": 62, "tax": 20.2},
    "baz": {"name": "Baz", "description": None, "price": 50.2, "tax": 10.5},
}

@app.get(
    "/items/{item_id}/name",
    response_model=Item,
    response_model_include={"name", "description"},
)
async def read_item_name(item_id: str):
    return items[item_id]

@app.get("/items/{item_id}/public", response_model=Item, response_model_exclude={"tax"})
async def read_item_public_data(item_id: str):
    return items[item_id]
```

## 技巧

`response_model_include` 和 `response_model_exclude` 接受一个 `set` 包含要包含或排除的字段名.

## 提示

你可以使用 `response_model_exclude_unset=True` 来只返回已设置的字段.
